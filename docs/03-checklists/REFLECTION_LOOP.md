# Reflection Loop

## Purpose
Provide a governed self-reflection and improvement loop for the platform.

## Reflection cadence
- after every major architecture change
- after every runtime failure
- after every production deploy
- after every important user workflow regression

## Required reflection questions
1. What changed?
2. What failed or nearly failed?
3. What assumptions were wrong?
4. What guardrail is missing?
5. What template or contract should be improved?
6. What can be automated next?

## Required outputs
- updated memory
- updated checklist
- updated contracts if needed
- updated issue ladder or next-step plan
