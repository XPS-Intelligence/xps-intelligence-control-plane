# XPS Master Request Compilation

## Objective
Compile the user’s platform requirements into a single implementation target.

## Immediate mandate
Build a hardened, Railway-first, GitHub-governed XPS Intelligence platform that:
- has a strong visual frontend
- operates reliably without collapsing under first use
- performs real lead ingestion and validation
- supports role-based workspaces
- supports AI assistants with memory, skills, and autonomy controls
- supports admin-level control and editing
- evolves into a repeatable SaaS and franchise-scale intelligence system

## Mandatory runtime capabilities
- Next.js frontend
- live API
- async worker system
- Redis-backed queues
- Postgres-backed canonical data
- CI/CD auto deploy
- Docker local runtime
- Railway production runtime
- health checks and postdeploy smoke

## Mandatory product capabilities
- employee page
- manager page
- owner page
- admin control page
- lead summary system
- lead scoring system
- lead validation system
- recommendation engine
- proactive agent messaging
- onboarding that seeds assistant memory
- autonomy mode settings
- prompt library
- template library
- skills library

## Mandatory intelligence capabilities
- industry taxonomy
- keyword and phrase discovery
- seed expansion
- public-web validation
- social scanning
- government/business-record scanning
- business status validation
- phone, address, and website freshness checks
- benchmarking
- trend signals
- pattern signals
- simulation and forecasting layer

## Mandatory admin/control-plane capabilities
- settings and policy control
- create/modify/delete for governed entities
- connector control
- source and seed control
- workflow and trigger control
- cost and health visibility
- prompt and skills governance
- allowed-list based agent tooling
- emergency shutoff

## Builder/editor direction
Use `open-lovable` as the primary donor for:
- center editor screen
- AI-assisted code and UI editing
- sandbox management
- preview and file inspection

Do not rebuild this surface from scratch if donor patterns can be transplanted.

## Runtime linkage target
`source adapters -> distallation -> xps-intel -> xps-intelligence-system -> CRM/operations`

## Success criteria
- honest runtime validation
- repeatable deployment
- low manual friction
- durable governance
- strong donor reuse
- clear path to passive intelligence and franchise-scale expansion
