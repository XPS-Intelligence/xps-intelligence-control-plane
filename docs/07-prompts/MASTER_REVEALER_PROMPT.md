# XPS Master Revealer Prompt

Use this prompt when you need the system to surface all relevant architecture, capabilities, risks, files, systems, and next actions without hiding complexity.

## Revealer invocation
Reveal the current XPS Intelligence platform truth at the highest practical fidelity.

For the active workspace:
- map every canonical repo and its role
- distinguish canonical repos from donor repos
- identify the active host runtime
- identify the intelligence and distillation path
- identify the current deployment, Docker, and CI/CD state
- identify what is implemented, what is stubbed, and what is missing
- identify current validation coverage and validation gaps
- identify auth, data, security, deployment, and observability risks
- identify waste, duplication, drift, and obsolete systems
- identify the fastest robust path to the next milestone

When revealing implementation strategy:
- prefer official and battle-tested open-source systems
- name the strongest template or reference architecture when relevant
- separate short-term MVP moves from long-term enterprise moves
- surface required env vars, ports, startup commands, health paths, data flows, and integration contracts
- provide exact file targets when implementation work is requested

When revealing gaps:
- be direct
- be specific
- do not soften blockers
- do not pretend something is validated if it is not
- do not fabricate integrations, benchmarks, or deployments

When revealing excellence:
- benchmark against the strongest public patterns from leading AI platform teams
- identify what those patterns do well
- identify where they remain weak for this product
- specify how XPS can be at least 10 percent better through domain grounding, safer autonomy, stronger validation, and tighter operations

When revealing next actions:
- prioritize actions that increase correctness, security, reliability, validation coverage, and deployability
- propose parallel lanes where they reduce total cycle time without causing merge chaos
