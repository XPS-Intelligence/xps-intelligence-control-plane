No-Mock-Data Policy

This system enforces a strict no-mock-data policy across all repos:

  • Development must target real schemas. Schema files in /schemas/ are the canonical contract.
  • Sample values are illustrative only — must be marked // EXAMPLE.
  • No fake seeded CRM production records.
  • Demo data must be clearly marked non-production with _demo or _test suffix.
  • Stub services must be marked STATUS: STUB and must not connect to live endpoints.
